JavaScript数据类型和变量(一)

发表于 2020-04-13 分类于 JavaScript
本文字数： 2.4k 阅读时长 ≈ 4 分钟

数据类型

计算机顾名思义就是可以做数学计算的机器，因此，计算机程序理所应当可以处理各种数值，但是远不止数值，还有文本、音频、视频、图片、网页等各种各样的数据，不同数据需要定义不同的数据类型，下面是JavaScript几类数据类型。

Number

JavaScript不区分整数和浮点数，统一用Number表示，以下是合法的Number

计算机由于是二进制的，所以有时候使用十六进制表示整数比较方便，十六进制使用0x做前缀，0-9，a-f的取值范围，例如0xff1,0x12f3,等等它和十进制表示的数值完全相同

Number可以直接使用四则运算

String

字符串以单引号’和双引号“括起来的任意文本，比如'abc',"abc"等等

Boolean

布尔值和布尔代数完全一致，一个布尔值只有true,false两种，要么是true，要么是false，他们可以直接使用，也可以通过布尔运算获得

&&代表与运算符，只有所有都为true才为true

||代表或运算符，只要有一个true就返回true

!非运算符，它是一个单目运算符，可以将true转换为false ,可以将false转换为true

布尔值常用的if判断

比较运算符

我们在使用对比运算符比较时，可以通过比较运算符获取布尔值

JavaScript支持任何数据类型比较

注意：==运算符，在JavaScript设计时，有两种比较运算符：

第一种==,它会自动转换数据类型再比较，很多时候会出现很多非常诡异的结果；

第二种===,他不会自动转换数据类型，如果类型不一致，直接返回false,如果一致在比较

由于JavaScript这个缺陷，不要使用==，坚持使用===比较

还有一个特殊的NaN,这个特殊的Number和任何一个值都不相同，包括他自己。

不够有一个唯一判断它的方法：isNaN();函数

注意：不要使用JavaScript进行浮点数运算，因为在浮点数运算过程种会产生误差，因为计算机无法精确表示无限循环小数，要是比较两个浮点数是否相等，只能计算他们之间的小数差的绝对值，看是否小于某个阈值

null和undefined

null表示一个”空“值，他和0和' '不同，0表示一个数值，而' '表示一个长度字符串为0，null则表示的是空值。

在JavaScript还有一个和null 类似的值，undefined它表示’未定义’，其实两者关系不大。所以，一般我们使用null，undefined，仅仅用于判断函数参数是否传递下用

Array

数组是一个有顺序的集合，集合中的值称为元素，在JavaScript中数组中可以是任何类型的值

上述包含7个元素，使用[]，将其包裹，使用,分隔

还有另一种方法Array()函数实现

在开发中我们长使用[]这种方法

Array中我们可以控制它的索引来访问它其中的元素，索引的初始值是0
